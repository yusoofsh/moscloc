## Direction Findings

- Defer shadcn RTL migration. The app displays Arabic text in places, but it is not currently an RTL shell; `src/index.css:143-145` handles Arabic text locally. The official shadcn CLI has an `rtl` migration, but applying it across full-screen display components would be a separate visual migration with higher review cost.
- Defer full-screen display card migration. `PrayerTimes`, `QuranVerse`, `CommunityEvents`, `AnnouncementBanner`, and `IqamahCountdown` use translucent, TV-style surfaces that are product-specific. Keep those custom until the shadcn foundation and admin migration are stable.

## Findings Considered And Rejected

- Replace the entire display UI with shadcn `Card` immediately: rejected for now because the display surfaces are intentionally immersive and verified by existing tests through `data-testid` and visible prayer-time content.
- Run `shadcn migrate rtl` now: rejected for now because the current app is Indonesian-first with localized Arabic text, not an RTL-first app shell.
